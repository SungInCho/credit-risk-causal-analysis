# Data

## Overview

This project uses the publicly available **Lending Club Loan Data** from Kaggle.
The raw CSV files are **not stored in this repository** (each is ~1.6–1.7 GB).
Place the downloaded files in `data/raw/` before running the notebooks.

---

## Download Instructions

1. Go to [https://www.kaggle.com/datasets/wordsforthewise/lending-club](https://www.kaggle.com/datasets/wordsforthewise/lending-club)
2. Sign in to Kaggle (free account required)
3. Click **Download** and unzip the archive
4. Place the two files in `data/raw/`:

```
data/raw/
├── accepted_2007_to_2018Q4.csv    (~1.6 GB, ~2.26M rows)
└── rejected_2007_to_2018Q4.csv   (~1.7 GB, ~27.6M rows)
```

---

## File Descriptions

| File | Rows | Description |
|---|---|---|
| `accepted_2007_to_2018Q4.csv` | ~2.26 M | All Lending Club loans that were funded (2007–2018 Q4). Contains loan terms, borrower attributes, and final loan status. |
| `rejected_2007_to_2018Q4.csv` | ~27.6 M | Loan applications that were rejected by LC's underwriting model. Contains limited borrower attributes and no outcome. |

After running the notebooks, the following intermediate files are saved to `outputs/intermediate/`:

| File | Generated by | Description |
|---|---|---|
| `accepted_cleaned.parquet` | `01_data_cleaning.ipynb` | Accepted loans filtered to 2013–2016 with cleaned dtypes and outliers handled |
| `accepted_with_current.parquet` | `01_data_cleaning.ipynb` | Same as above but includes 'Current' loans (used in Notebook 05 robustness check) |
| `rejected_cleaned.parquet` | `01_data_cleaning.ipynb` | Rejected loans filtered to 2013–2016 with basic cleaning applied |
| `accepted_modeling.parquet` | `02_eda_and_features.ipynb` | Final modeling dataset with engineered features, treatment variables, and one-hot encodings |
| `meta.json` | `02_eda_and_features.ipynb` | Column lists (covariates, grade order, default statuses) used consistently across notebooks |

---

## Sample Period

The analysis is restricted to **2013–2016 issue years** for the following reasons:

- **36-month loans** issued through 2016 matured by early 2019, well before the 2018 Q4 data cut-off — the vast majority of loans in this window have a resolved outcome (Fully Paid or Charged Off).
- **2017+ loans**: many are still labeled 'Current' at the cut-off date, which would downward-bias default-rate estimates.
- **Pre-2013**: smaller volume and structurally different market conditions (post-GFC recovery period).

## Key Variables Used

| Variable | Type | Description |
|---|---|---|
| `int_rate` | Float | Annual interest rate (%) |
| `grade` / `sub_grade` | Categorical | LC credit tier (A–G, with 5 sub-tiers each) |
| `loan_status` | Categorical | Final outcome: Fully Paid, Charged Off, Default, Current, etc. |
| `fico_range_low/high` | Integer | FICO score range reported at origination |
| `dti` | Float | Debt-to-income ratio (%) |
| `annual_inc` | Float | Self-reported annual income ($) |
| `loan_amnt` | Float | Funded loan amount ($) |
| `term` | String | '36 months' or '60 months' |
| `purpose` | Categorical | Stated loan purpose (debt consolidation, home improvement, etc.) |
| `revol_util` | Float | Revolving credit utilisation rate (%) |
| `emp_length` | String | Employment length category |
| `issue_d` | Date | Loan origination month |
